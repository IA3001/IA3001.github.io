1004 搭积木

Problem Description

小 W 正在用积木搭建一个巨大的结构。

有 nnn 块积木，编号为 111 到 nnn。第 iii 块积木有两个整数属性 aia_iai​ 和 bib_ibi​。

一开始，小 W 打算一块一块地搭建。对每块积木 iii，给定一个积木 fif_ifi​，表示积木 iii 必须直接放在积木 fif_ifi​ 的上方。如果 fi=0f_i=0fi​=0，则积木 iii 位于整个结构的最底部。题目保证这些要求是自洽的，并且最终会形成一个包含所有积木的完整结构。

后来，小 W 觉得逐块搭建太无聊了。他想到一种新方法：可以先把若干块积木组装成一个结构，再一次性把整个已组装结构放到另一个结构上。具体地，对于两个不交的连通集合 AAA 和 BBB，若 AAA 中存在唯一的点 xxx 使得 fx∈Bf_x \in Bfx​∈B，那么 AAA 就可以被放在 BBB 的上面。

也就是说，在满足所有原始直接上下关系的前提下，小 W 可以自由选择执行放置操作的顺序，也可以提前组装某些局部结构。

一次放置操作中，假设一个已经组装好的结构被放到另一个结构的上方。令：

AAA 为上方结构内所有积木的 aia_iai​ 之和；

BBB 为下方承托结构内所有积木的 bib_ibi​ 之和。

这次放置操作的代价定义为 A×BA \times BA×B。

操作完成后，两个结构会合并为一个更大的结构。

注意，下方承托结构指的是当前已经组装在一起的整个结构，而不只是单块积木或它的祖先。

小 W 想知道，在所有合法的搭建顺序中，最小总代价是多少。

Input

本题单个测试点内有多组测试数据。

第一行输入一个正整数 TTT (1≤T≤20)(1 \le T \le 20)(1≤T≤20)，表示测试数据组数。

接下来按如下格式输入 TTT 组数据：

第一行输入一个整数 nnn (1≤n≤2×105)(1 \le n \le 2 \times 10^5)(1≤n≤2×105)，表示积木数量。保证单个测试点内所有测试数据的 nnn 之和不超过 10610^6106。

第二行输入 nnn 个整数 a1,a2,…,ana_1, a_2, \dots, a_na1​,a2​,…,an​ (1≤ai≤103)(1 \le a_i \le 10^3)(1≤ai​≤103)。

第三行输入 nnn 个整数 b1,b2,…,bnb_1, b_2, \dots, b_nb1​,b2​,…,bn​ (1≤bi≤103)(1 \le b_i \le 10^3)(1≤bi​≤103)。

第四行输入 nnn 个整数 f1,f2,…,fnf_1, f_2, \dots, f_nf1​,f2​,…,fn​。其中 f1=0f_1=0f1​=0，对于所有 i>1i>1i>1，保证 1≤fi<i1 \le f_i < i1≤fi​<i。

Output

对于每组数据，输出一行一个整数，表示搭建完整结构所需的最小总代价。

Sample Input

1
3
3 10 1
5 1 10
0 1 1

Sample Output

56

Hint

样例中，积木 222 和 333 都必须直接放在积木 111 上方。

如果先把积木 222 放到积木 111 上，代价为 10×5=5010 \times 5 = 5010×5=50。

此时下方结构包含积木 {1,2}\{1, 2\}{1,2}，其 bbb 之和为 5+1=65 + 1 = 65+1=6。再放置积木 333，代价为 1×6=61 \times 6 = 61×6=6。

总代价为 50+6=5650 + 6 = 5650+6=56，可以证明这是最优的。

本题输入量较大，建议使用较快速的输入方式（如关闭流同步的 cin）。

Solution

类树上 Smith rule 合并贪心

先合并谁的问题，当一个父节点 P(a,b)P(a, b)P(a,b) 和子节点 X(a1,b1)X(a_1, b_1)X(a1​,b1​) 与 Y(a2,b2)Y(a_2, b_2)Y(a2​,b2​) 合并时，可以先合并 XXX 当且仅当

a1b+a2(b+b1)<a2b+a1(b+b2)⇒a2b1<a1b2a_1b + a_2(b+b_1) < a_2b + a_1(b+b_2) \\ \Rightarrow \\ a_2b_1 < a_1b_2 a1​b+a2​(b+b1​)<a2​b+a1​(b+b2​)⇒a2​b1​<a1​b2​

#include<bits/stdc++.h>
using namespace std;
using LL = long long;

const int N = 2e5 + 5;

void write(__int128 x) {
    if (x < 0) x = -x, putchar('-');
    if (x == 0) putchar('0');
    stack<char> S;
    while(x) S.push('0' + x % 10), x /= 10;
    while(S.size()) putchar(S.top()), S.pop();
    putchar('\n');
}

struct DSU {
    vector<int> f;
    void init(int n) {
        f = vector<int>(n);
        iota(f.begin(), f.end(), 0);
    }
    int find(int x) {
        return f[x] == x ? x : f[x] = find(f[x]);
    }
    bool merge(int x,int y) {
        x = find(x), y = find(y);
        assert(x != y);
        if (x == y) return false;
        f[y] = x;
        return true;
    }
}D;

int T;
int t[N];

struct Node{
    LL a, b, u, t;
    bool operator<(const Node& o) const {
        return b*o.a != a*o.b ? b*o.a > a*o.b : t > o.t;
    }
};

int n;
int a[N], b[N], f[N];
priority_queue<Node> Q;

void solve() {
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> a[i];
    for (int i = 1; i <= n; i++) cin >> b[i];
    for (int i = 1; i <= n; i++) cin >> f[i];

    D.init(n + 1);
    for (int i = 1; i <= n; i++) Q.emplace(a[i], b[i], i, t[i] = ++T);
    
    __int128 ans = 0;
    while(Q.size()) {
        auto [_a, _b, u, _t] = Q.top(); Q.pop();
        if (t[u] != _t) continue;
        if (f[u] == 0) continue;
        int ff = D.find(f[u]);
        if(D.merge(ff, u)) {
            ans += _a * b[ff];
            a[ff] += _a;
            b[ff] += _b;
            Q.emplace(a[ff], b[ff], ff, t[ff] = ++T);
        }
    }
    write(ans);
}

int main() {
    ios::sync_with_stdio(0); cin.tie(0); cout.tie(0);
    int T = 1;
    cin >> T;
    while (T--) solve();
}

1010 游戏

Problem Description

Alice 和 Bob 在玩取石子游戏，有 nnn 堆石子从左到右排成一排，初始时从左到右第iii 堆有 xix_ixi​ 颗石子。

Alice 和 Bob 轮流操作：

  • 轮到 Alice 操作时，Alice 从最左边的堆选至少一个石子，把选中的石子移到第二左的堆。
  • 轮到 Bob 操作时，Bob 从最右边的堆选至少一个石子，把选中的石子移到第二右的堆。

若轮到某人无法操作时，当前操作的人输掉游戏。

若 Alice 和 Bob 都使用最优策略，你需要判断 Alice 是否有先手必胜策略。

Input

第一行输入一个正整数 T(1≤T≤105)T(1 \le T \le 10^5)T(1≤T≤105)，表示数据组数。接下来按如下格式输入 TTT 组数据：

每组第一行输入一个整数 n(1≤n≤106)n(1 \le n \le 10^6)n(1≤n≤106)。

第二行输入 nnn 个正整数表示每堆石子数 x1,x2,…,xn(1≤xi≤109)x_1,x_2,\dots,x_n(1 \le x_i \le 10^9)x1​,x2​,…,xn​(1≤xi​≤109)。

保证输入数据中 ∑n≤3×106\sum n \le 3 \times 10^6∑n≤3×106。

Output

共输出 TTT 行。

如果 Alice 有先手必胜策略，输出 YES，否则输出 NO。

Solution

从 基础情况 建立 必胜/必败 判断体系，找规律

  • n=1n = 1n=1 时，必败
  • n=2n = 2n=2 时，必胜
  • n=3n=3n=3 时，设为 a,b,ca,b,ca,b,c
    • Alice 不可能全挪，否则必败
    • Bob 同样不可能全挪
    • 只能看谁先挪空⇒a>c时必胜，否则必败只能看谁先挪空 \Rightarrow a > c 时必胜，否则必败只能看谁先挪空⇒a>c时必胜，否则必败
    • 即轮到 Alice 并且只剩下 333 堆的话，充要条件是 a>ca > ca>c
  • n=4n=4n=4 时，设为 a,b,c,da,b,c,da,b,c,d
    • 若 a+b≥da+b \ge da+b≥d ，Alice 可以全挪，到达 n=3n=3n=3 必胜
    • 若 a+b<da+b \lt da+b<d ，Alice 全挪必败，部分挪则 Bob 可以全挪之后 n=3n=3n=3 ，因为必定 a−x<c+da-x<c+da−x<c+d 所以必败
    • 充要条件是 a+b≥da+b \ge da+b≥d
  • n=5n=5n=5 时，设为 a,b,c,d,ea,b,c,d,ea,b,c,d,e
    • 若 a+b>d+ea+b > d+ea+b>d+e ，Alice 全挪，到达 n=4n=4n=4 必胜
    • 若 a+b<d+ea+b<d+ea+b<d+e ，Alice 全挪必败，故部分挪，此时 Bob 全挪，到达 n=4n=4n=4 ，因为 a+b<(d+e)a+b<(d+e)a+b<(d+e) 所以必败
    • 若 a+b=d+ea+b=d+ea+b=d+e ，Alice 全挪必败，故部分挪，同样，Bob 全挪必败，故也部分挪，直到某一个人挪动导致 n=4n=4n=4 那么另一方就来到 n=4n=4n=4 的情况直接胜，故充要条件是 a>ea>ea>e
  • n>5n>5n>5 时，似乎可以总结出来规律
    • 若 prei>sufjpre_i > suf_jprei​>sufj​ 必胜
    • 若 prei<sufjpre_i < suf_jprei​<sufj​ 必败
    • 若 prei=sufjpre_i = suf_jprei​=sufj​ 未决定，需要比较 prei−1pre_{i-1}prei−1​ 和 sufj+1suf_{j+1}sufj+1​
    • 为什么？
      • 归纳假设，挖掉中间的一个，左右两边分别是 preprepre 和 sufsufsuf
      • n+1n+1n+1 的比较，必然可以基于 nnn 的比较，不受先手后手的影响
      • 针对于相等情况，这与先后手有关，先手必败，故需要尽量切换先后手顺序
        • 注意：切换先手后手顺序，其实是比较最外层 已经累计得到 的数量（prei−1>sufj+1pre_{i-1} > suf_{j+1}prei−1​>sufj+1​）

#include<bits/stdc++.h>
using namespace std;
using LL = long long;

const int N = 1e6 + 5;

LL n, a[N], pre[N], suf[N];

void solve() {
	cin >> n;
	suf[n + 1] = 0;
	for (int i = 1; i <= n; i++) cin >> a[i], pre[i] = pre[i - 1] + a[i];
	for (int i = n; i >= 1; i--) suf[i] = suf[i + 1] + a[i];
	if (n == 1) return void(cout << "NO\n");
	if (n == 2) return void(cout << "YES\n");
	for (int m = n / 2 + 1, i = m - 1, j = m + 1; j <= n; i--, j++) {
		if (pre[i] > suf[j]) return void(cout << "YES\n");
		else if (pre[i] < suf[j]) return void(cout << "NO\n");
	}
	if (n & 1) cout << "NO\n";
	else cout << "YES\n";
}
int main() {
	ios::sync_with_stdio(0); 
	cin.tie(0); cout.tie(0);
	int T = 1;
	cin >> T;
	while (T--) solve();
}
