B 再买一瓶

Problem Description

一瓶饮料售价 111 元。

每次小明喝完一瓶饮料，他会以 ab\dfrac{a}{b}ba​ 的概率获奖。如果获奖，他将得到 ccc 元；否则，他什么也得不到。

他得到的钱可以用来购买更多饮料。每瓶饮料的获奖事件相互独立。

小明一开始有 nnn 元。求他恰好喝完 mmm 瓶饮料后花光所有钱的概率。

Input

每个测试文件包含多组测试数据。第一行包含测试数据的组数 TTT (1≤T≤200 0001 \le T \le 200\,0001≤T≤200000)。

接下来 TTT 行，每行包含五个整数 n,m,c,a,bn, m, c, a, bn,m,c,a,b (1≤n,m,c≤2 000 0001 \le n, m, c \le 2\,000\,0001≤n,m,c≤2000000，0≤a<b<998 244 3530 \le a < b < 998\,244\,3530≤a<b<998244353)，表示小明一开始有 nnn 元，每次获奖得到 ccc 元，获奖概率为 ab\dfrac{a}{b}ba​；你需要求出他恰好喝完 mmm 瓶饮料的概率。

Output

输出 TTT 行，每行包含一个整数：对应测试数据的答案对 998 244 353998\,244\,353998244353 取模的结果。

形式化地，设 M=998 244 353M = 998\,244\,353M=998244353。可以证明，答案可以表示为最简分数 pq\dfrac{p}{q}qp​，其中 ppp 和 qqq 为整数且 q≢0(modM)q \not\equiv 0 \pmod{M}q≡0(modM)。输出等于 p⋅q−1 mod Mp \cdot q^{-1} \bmod Mp⋅q−1modM 的整数。

Sample Input

4
1 1 2 1 2
1 3 2 1 2
2 2 3 1 3
1 2 2 1 2

Sample Output

499122177
873463809
776412275
0

Hint

在第一组测试数据中，恰好喝完 111 瓶的唯一方式是第一瓶没有获奖，其概率为 12\dfrac{1}{2}21​。

在第二组测试数据中，恰好喝完 333 瓶的唯一方式是第一瓶获奖且接下来两瓶都没有获奖，其概率为 18\dfrac{1}{8}81​。

在第四组测试数据中，小明一开始有 111 元，每次获奖得到 222 元，因此他不可能恰好喝完 222 瓶。

Solution

Raney 引理：投票问题的结论，从 000 到 nnn，最大上升步长 ≤1\le 1≤1，符合 前缀约束 的序列数量的结论

Raney 引理：若整数序列每一项不超过 111（ai≤1a_i \le 1ai​≤1） ，总和为正整数 nnn（n≥1n \ge 1n≥1） ，则它的 mmm （m=∑i=1kwim = \sum_{i=1}^k w_im=∑i=1k​wi​） 个循环移位中恰有 nnn 个移位的所有非空前缀和为正。

∑i=1kaiwi∑i=1kwi×(∑i=1kwiw1,w2,...,wk)\frac{\sum_{i=1}^k a_i w_i}{\sum_{i=1}^k w_i} \times \binom{\sum_{i=1}^k w_i}{w_1, w_2, ..., w_k} ∑i=1k​wi​∑i=1k​ai​wi​​×(w1​,w2​,...,wk​∑i=1k​wi​​)

Code

#include<bits/stdc++.h>
using namespace std;

const int N = 2e6 + 5, mod = 998244353;

int n, m, c, a, b;
int fac[N], invfac[N];

int ksm(int a, int b = mod - 2) {
	int res = 1;
	for (; b; b >>= 1, a = 1LL * a * a % mod) if (b & 1) res = 1LL * res * a % mod;
	return res;
}

void init() {
	fac[0] = 1;
	for (int i = 1; i < N; i++) fac[i] = 1LL * fac[i - 1] * i % mod;
	invfac[N - 1] = ksm(fac[N - 1]);
	for (int i = N - 1; i >= 1; i--) invfac[i - 1] = 1LL * invfac[i] * i % mod;
}

int C(int i, int j) {
	if (i < j) return 0;
	return 1LL * fac[i] * invfac[j] % mod * invfac[i - j] % mod;
}

void solve() {
	cin >> n >> m >> c >> a >> b;
	if (m - n < 0 || (m - n) % c) {
		cout << 0 << "\n";
		return;
	}
	int k = (m - n) / c;
	int p = 1LL * a * ksm(b) % mod;
	int q = (1 - p + mod) % mod;
	cout << 1LL * n * ksm(m) % mod * C(m, k) % mod * ksm(p, k) % mod * ksm(q, m - k) % mod << "\n";
}

int main() {
	init();
	ios::sync_with_stdio(0); cin.tie(0); cout.tie(0);
	int T = 1;
	cin >> T;
	while (T--) solve();
}

G 矩阵标记

Problem Description

给定一个 n×mn \times mn×m 的数字网格，每个格子中包含一个正整数。

如果存在两个格子 (r1,c1)(r_1, c_1)(r1​,c1​) 和 (r2,c2)(r_2, c_2)(r2​,c2​) 满足 r1<r2r_1 < r_2r1​<r2​ 且 c1<c2c_1 < c_2c1​<c2​，且这两个格子中的数字相等，那么所有满足 r1≤r≤r2r_1 \le r \le r_2r1​≤r≤r2​ 且 c1≤c≤c2c_1 \le c \le c_2c1​≤c≤c2​ 的格子 (r,c)(r, c)(r,c) 都会被标记。

输出整个网格最终的标记情况。

Input

每个测试文件仅包含一组测试数据。

第一行包含两个整数 n,mn, mn,m (1≤n,m1 \le n, m1≤n,m，n×m≤1 000 000n \times m \le 1\,000\,000n×m≤1000000)。

接下来 nnn 行，每行包含 mmm 个整数；其中第 iii 行包含 ai,1,ai,2,…,ai,ma_{i,1}, a_{i,2}, \dots, a_{i,m}ai,1​,ai,2​,…,ai,m​ (1≤ai,j≤n×m1 \le a_{i,j} \le n \times m1≤ai,j​≤n×m)，描述该数字网格。

Output

输出 nnn 行，每行包含一个长度为 mmm、由字符 0 和 1 组成的字符串。

第 iii 行的第 jjj 个字符为 1 表示格子 (i,j)(i, j)(i,j) 最终被标记，为 0 表示未被标记。

Sample Input

2 6
1 2 3 4 5 6
7 8 9 3 4 10

Sample Output

001110
001110

Hint

在样例中，第 1 行第 3 列的格子和第 2 行第 4 列的格子都含有数字 3，因此它们标记了两行中第 3 列到第 4 列的所有格子。

第 1 行第 4 列的格子和第 2 行第 5 列的格子都含有数字 4，因此它们标记了两行中第 4 列到第 5 列的所有格子。两个矩形的并集即为最终答案。
